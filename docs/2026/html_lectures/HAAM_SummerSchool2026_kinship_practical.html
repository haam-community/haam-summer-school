READv2 analysis

Since we already have READv2 in our environment, you do not need to clone the repository from GitHub.

Extract the individuals that we selected for kinship analysis from the dataset using the Poseidon framework (Poseidon trident tool tutorial)

trident forge -d /vol/volume/packages-poseidon/day3_pkg \
  -f "<RISE1160_noUDG.SG>,<RISE1161_noUDG.SG>,<RISE1164_noUDG.SG>,<RISE1168_noUDG.SG>,<RISE1169_noUDG.SG>" \
  -o /vol/volume/genetic-kinship/practical/genotype_data/Schroeder_Poland_subset \
  --outFormat PLINK
  
  #forge       : subselecting individuals 
  #-d          : the path to the dataset
  #-f          : individual IDs
  #-o          : output file name prefix
  #--outFormat : output file format

Alternatively, you can run PLINK to subset individuals using the “—keep” option (Input filtering - PLINK 1.9).

Extract autosomes from the dataset using PLINK

plink --bfile /vol/volume/genetic-kinship/practical/genotype_data/Schroeder_Poland_subset --autosome --make-bed --out /vol/volume/genetic-kinship/practical/genotype_data/Schroeder_Poland_subset/Schroeder_Poland_subset_auto

#--bfile   : prefix for name of the file in PLINK format (bed/bim/fam)
#--autosome: keep only autosomes (removing sex chromosomes)
#--make-bed: creating PLINK files
#--out     : output file name prefix

Run READv2

cd /vol/volume/genetic-kinship/practical/READv2
kinship-read --i /vol/volume/genetic-kinship/practical/genotype_data/Schroeder_Poland_subset/Schroeder_Poland_subset_auto

#--i : prefix for name of the file in PLINK format (bed/bim/fam)

The default normalization method is median. Other normalization methods (maximum, mean, or user-defined value) can be specified using the “-n” option. The “--norm_value” option should be included to set the normalization value manually.

KIN analysis

Downsample two BAM files with larger sizes to save space and time using samtools (samtools-view(1) manual page). 

 cd /vol/volume/genetic-kinship/data
 samtools view -b -s 0.10 Koszyce-3_10_final.bam > Koszyce-3_10_final_subset10.bam
 samtools view -b -s 0.10 Koszyce-3_2_final.bam > Koszyce-3_2_final_subset10.bam
 
 #-b : output in BAM format
 #-s : subsampling specified by a float indicating the proportion of alignments to be kept

You do not need to index your BAM files before running KIN, as KINgaroo does indexing and sorting by default. You can skip these steps by using the “-s” parameter. The parameter can be set to zero to save time if your BAM files have already been indexed and sorted.

Create a bed file including SNP positions (only autosomes) and reference/alternative alleles

cd /vol/volume/genetic-kinship/practical/KIN
awk '{print $1,$4-1,$4,$5,$6}' /vol/volume/genetic-kinship/practical/genotype_data/Schroeder_Poland_subset/Schroeder_Poland_subset.bim | sed '/^23/d' | sed '/^24/d' | tr ' ' '\t' > schroeder_poland_positions.bed

Run KINgaroo and then KIN

cd /vol/volume/genetic-kinship/practical/KIN
KINgaroo -bam /vol/volume/genetic-kinship/data -bed schroeder_poland_positions.bed -T targets.txt -cnt 0 > king.log

#-bam : BAM file directory
#-bed : bed file including positions and reference/alternative alleles
#-T   : target file of BAM file name prefices
#-cnt : contamination correction (zero indicating no correction)

KIN -I /vol/volume/genetic-kinship/practical/KIN/ -O /vol/volume/genetic-kinship/practical/KIN/analysis/ > kin.log

#-I : directory where the KINgaroo results saved
#-O : output directory

Before running KINgaroo, do not forget to check that the BAM files are in the same directory!
While running KIN, keeping the “/” at the end of paths is critical. Otherwise, you would get an error. Also, you do not need to specify the target file anymore.

Remember to delete unnecessary files from your directory at the end of the analysis. After running KINgaroo and verifying your input files (CSV files starting with “input”), you can safely remove the /splitbams and /bedfiles directories, as they are specifically needed only for the KINgaroo process.

#Copy the following R code block into your plot.R file
library(tidyverse)

#Read the results and the file of individual IDs
kin <- read.csv("/vol/volume/genetic-kinship/practical/KIN/analysis/KIN_results.csv",header=TRUE,stringsAsFactors = FALSE,sep="\t")

readv2 <- read.table("/vol/volume/genetic-kinship/practical/READv2/Read_Results.tsv",header=TRUE,stringsAsFactors = FALSE,sep="\t")

indv <- read.table("/vol/volume/genetic-kinship/practical/visualization/indv",header=FALSE,stringsAsFactors = FALSE)

# Function to create a standardized pair string (alphabetically ordered).This ensures that pairs like "A,B" and "B,A" are treated the same.
sort_pair <- function(pair) {
  inds <- unlist(strsplit(pair, ","))
  paste(sort(inds), collapse = ",")
}

# Process KIN dataset
# Calculate kinship coefficient from KIN results and create a column of pairs ID to mimic the column of pairs ID in Readv2 results 
kin <- kin %>%
  #Separate the Pair column into individual IDs
  separate(Pair,sep = "_\\._",into = c("Ind1","Ind2"),remove = FALSE) %>%
  # Map original IDs (Ind1, Ind2) to new IDs using the 'indv' lookup table
  mutate(Ind1 = indv$V2[match(Ind1, indv$V1)],Ind2 = indv$V2[match(Ind2, indv$V1)],
  # Create a readable pair string for later reference
  PairIndividuals = paste0(Ind1, ",", Ind2),
  # Calculate kinship coefficient theta from k1 and k2 values
  KIN_Theta=k1/4+k2/2)
  
#Subset relevant columns for merging later
kin_theta <- kin %>%
  select(PairIndividuals, KIN_Theta)

# Process READv2 dataset
#Subset the columns of READv2 result file and rename the KinshipCoefficient column to be consistent
readv2_theta <- readv2 %>%
  select(PairIndividuals,KinshipCoefficient) %>%
  rename(Readv2_Theta = KinshipCoefficient) %>%
  # Set negative kinship coefficient values to zero (indicating unrelatedness)
  mutate(Readv2_Theta = ifelse(Readv2_Theta < 0, 0, Readv2_Theta))

# Standardize pair IDs for both datasets
kin_theta <- kin_theta %>%
  mutate(Pair_std = sapply(PairIndividuals, sort_pair))

readv2_theta <- readv2_theta %>%
  mutate(Pair_std = sapply(PairIndividuals, sort_pair))

# Merge datasets by standardized pair ID 
merged <- readv2_theta %>%
  # Keep PairIndividuals and Readv2_Theta from readv2_theta
  select(PairIndividuals, Readv2_Theta, Pair_std) %>%
  # Join corresponding KIN_Theta values by Pair_std
  left_join(kin_theta %>% select(Pair_std, KIN_Theta), by = "Pair_std")

# Reshape data for plotting/comparison
final_data <- merged %>%
  # Pivot the theta columns into long format with columns Tool and Theta
  pivot_longer(
    cols = c(Readv2_Theta, KIN_Theta),
    names_to = "Tool",
    values_to = "Theta"
  )

# Now 'final_data' has four columns: PairIndividuals, Pair_std, Tool (READv2 or KIN), Theta

#Generate a data frame for geom_text() to label the horizontal lines of theoretical kinship values
x<-9.5
y<-c(0.245,0.12,0.058,-0.005)
labels<-c("first degree","second degree","third degree","unrelated")
plot.labels<-data.frame(x,y,labels)

#Plot the results
plot_kinship <- final_data %>%
  ggplot() +
  # Add horizontal reference lines for theoretical kinship values
  geom_hline(aes(yintercept=0.25,linetype="First Degree"),col="black")  + 
  geom_hline(aes(yintercept=0.0,linetype="Unrelated"),col="black") + 
  geom_hline(aes(yintercept=0.125,linetype="Second Degree"),col="black")+ 
  geom_hline(aes(yintercept=0.0625,linetype="Third Degree"),col="black")+
  geom_point(aes(x=reorder(PairIndividuals,Theta),y=Theta,color=Tool,shape=Tool,fill=Tool),size=5.5) +
  # Add text labels for theoretical lines
  geom_text(data=plot.labels, aes(x=x, y=y, label=labels),colour="#481d24",size=4,fontface="italic") +
  scale_shape_manual(name = "Tool",
                     labels = c("READv2","KIN"),
                     values = c(19,17),
                     limits=c("Readv2_Theta","KIN_Theta")) + 
  scale_colour_manual(values=c("#7d84b2","#ad336d"),
                      labels = c("READv2","KIN"),
                      limits=c("Readv2_Theta","KIN_Theta")) +
  scale_fill_manual(values=c("#7d84b2","#ad336d")) +
  scale_linetype_manual(values = c(5,6,4,3)) +
  # Set y-axis breaks and labels
  scale_y_continuous(breaks = c(0,0.0625,0.125,0.25),labels=c("0.0","0.0625","0.125","0.25"))+
  labs(title="Genetic relatedness estimation for Koszyce samples",x="",
       y = "Kinship Coefficient") +
  theme_bw() +
  theme(
        axis.text.x = element_text(angle = 90,size=10,face="bold"),
        axis.title.y = element_text(size=12,face="bold"),
        axis.text.y = element_text(size=10,face="bold"),
        panel.grid.major = element_blank(), panel.grid.minor = element_blank(),
        panel.background = element_blank(), axis.line = element_line(colour = "black"),
        plot.title = element_text(face = "bold"),
        legend.title = element_text(size=15,face="bold"),
        legend.text = element_text(size=15)
        )+
  # Hide fill, linetype, and shape guides in the legend
  guides(fill = "none",linetype="none",shape="none")
